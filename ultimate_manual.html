входной файл

md_modules

Содержит основные модули, все что нужно для молекулярной динамики, кроме функций относящихся к взаимодействию частиц (расчет сил и энергии взаимодействия), эти функции содержутся в interaction_modules.

md_general

types in md_general

Производные типы данных для молекулярной динамики.

type time_steps

Содержит величину одного шага по времени, половины шага, четверти шага и одной восьмой.

type simulation_cell

Вычислительная ячейка. Содержит размеры вычислительной ячейки по X,Y и Z.

type particles

Частицы. Содержит информацию о частицах: количество, координаты, скорости, массы, силы, названия.

type particle_group

Группа частиц. Содержит количетво частиц и их индексы частиц в массивах внутри type particles.

type nose_hoover_chain

Цепочка термостатов Нозе-Гувера. Содержит количество звеньев, количество степеней свободы подключенной системы частиц, координаты звеньев, скорости звеньев, массы звеньев, заданную температуру, шкалирующий множитель и полную энергию цепочки термостатов.

чтото там как то настраивается

type neibour_list

Списки соседей. Содержит количество соседей, номера в type neibour_list?, расстояния до соседей (векторы и их модули), индексы частиц в массивах внутри type particles, количество списков, максимальное количетво соседей, частоту обновления списков, радиус обрезания взаимодействия.

!!!

type integrator_params

Общие для всех интеграторов параметры. Содержит название интегратора, шаг по времени, количество шагов (сколько он будет работать), частоты для записи ???, ???

Подпрограммы для молекулярной динамики, анализа положения частиц и всего что угодно.

Выводит информацию о группе частиц в .xyz файл.

subroutine integrate_verlet_positions(atoms,group,steps,box)

Изменяет положения частиц (за один временной шаг) согласно алгоритму интегрирования уравнений движения Верле в скоростной форме.

subroutine integrate_verlet_velocities(atoms,group,steps)

Изменяет скорости частиц (за половину временного шага) согласно алгоритму интегрирования уравнений движения Верле в скоростной форме.

subroutine molecular_static_velocities(atoms,group)

Изменяет скорости частиц. От скорости остается компонента параллельная силе, действующей на частицу. Если эта компонента направлена в противоположную сторону с силой, то скорость приравнивается к нулю.

subroutine zero_forces(atoms,group)

Обнуляет силы.

subroutine scale_velocities(atoms,group,s)

Умножает все скорости на s.

subroutine random_velocities(atoms,group)

Присваивает проекциям скоростей частиц случайные величины согласно распределению Гаусса, с нулевым среднем и ? дисперсией.

subroutine random_momenta(atoms,group)

Присваивает проекциям скоростей частиц случайные величины согласно, так чтобы проекции импульсов имели гауссовское распределение. ? ?

subroutine position_analysis(av,mi,ma,atoms,group,direction,minimum,maximum)

subroutine find_distance(dr,vec1,vec2,box)

Находит рассояние между двумя точками с учетом периодических граничных условий по X, Y и Z.

Производит шаг интегрирования цепочки термостатов Нозе-Гувера, корректирует скорости частиц из группы. Таких шагов два в одном шаге МД.

subroutine calculate_nose_hoover_chain_energy(nhc)

subroutine create_neibour_list(nl)

Выделяет память под списки соседей.

subroutine update_neibour_list(md_step,nl,atoms,group1,group2,box)

Обновляет списки соседей. Пересчитывает взаимное расположение соседей, если нужно ищет соседей заново согласно частоте обновления списков.

subroutine find_neibours(nl,atoms,group1,group2,box)

Для каждого атома из group1 находит соседей из group2 (порядок важен!). group1 и group2 могут совпадать.

subroutine find_neibour_distances(nl,atoms,group1,group2,box)

Обновляет списки соседей. Пересчитывает взаимное расположение соседей. nl, group1 и group2 должны быть те же что и в subroutine find_neibours.

subroutine converce_neigbour_list(cnl,group,nl)

Пересчитывает из списков соседей nl найденных для group1 и group2 в списки cnl для group2 и group1.

md_interactions

Модуль содержит подпрограммы зависящие от потенциалов взаимодействия частиц. Сами потенциалы описываются отдельно.

types in md_interactions

type interaction_parameters

Парметры взаимодействия. Тип данных может хранить сколько угодно, каких угодно параметров взаимодействия. В текущей реализации он хранит только один набор параметров для кагого либо взаимодействия.

type interaction

Описывает взаимодействие. Содержит имя взаимодействия, имя файла с параметрами, параметры взаимодействия, энергию взаимодействия, номера взаимодействующих групп частиц из входного файла, необходимое количество списков соседей и максимальный порядок соседей, с которыми взаимодействует частица (насколько далеких соседей надо учесть чтобы найти силу, действующую на частицу).

Добавляет к текушим силам частиц силы от всех взаимодействий. Считает силы аналитически используя соответствующие подпрограммы из модулей для каждого взаимодействия.

subroutine energy(inter_name,e,nl,p)

Находит энергию взаимодействия частиц по спискам nl, для взаимодействия с именем inter_name.

subroutine calculate_potential_energies(interactions)

Находит суммарную энергию взаимодействия от всех взаимодействий.

subroutine calculate_forces_numerically(atoms,interactions)

Считает силы численно, используя только зависимость энергии взаимодействия от расположения частиц. Не дописано для некоторых взаимодействий.

md_simulation

subroutine md(out_id,all_out_id,input_path,settings_filename,output_prefix,out_period,num_of_omp_treads)

Собственно алгоритм молекулярной динамики. В данной реализации может моделировать NVE и NVT ансамбли, а также минимизировать потенциальную энергию методом молекулярной статики. Для NVE ансамля используется скоростной алгоритм Верле, для NVT - цепочка термостатов Нозе-Гувера.
